You ever built something and then thought, “Wow, I don’t even remember what I learned?”

I do. A lot.

I’ll get deep into a project, solve the problem, push the commit, and move on. A few weeks later the actual story is already fuzzy.

The repo exists. The specs exist. The commits exist. Maybe there’s a README at the end, if I remembered to write one.

But the real part usually gets buried somewhere private.

The reason I started. The false starts. The decisions. The moments where I realized I was overengineering. The little wins. The things I misunderstood. The things I learned.

That felt wrong.

A lot of my side projects are not just attempts to make products. They are how I learn. They are how I practice architecture, security, DevOps, AI-assisted development, writing, design, and eventually marketing. They are also part of my professional story.

If I am going to spend this much time building things, I want the work to have a chance at seeing the light of day.

That is the motivation behind Launch Start.

I want this to become the home for my side projects: a place where each project has an overview, screenshots, links, notes, and a record of how it was built.

Not just a polished “look what I made” page.

More like a build journal.

At first, I imagined Launch Start as a 30-day or 90-day sprint challenge site. That framing is still useful, but I do not want the whole thing locked to a timer. Some projects might be short. Some might be long. Some might be experiments. Some might become real products.

The duration is not the point.

The point is that the work has a beginning, a purpose, a visible trail, and a reflection at the end.

So the first project on Launch Start is Launch Start itself.

That feels fitting.

Before I can document the other projects, I need to build the place where the documentation lives.

The goal is simple: create a public home for my side projects, my writing, and my engineering growth.

If this works, Launch Start becomes more than a portfolio.

It becomes a record of me learning how to build things properly — in public, with receipts.
